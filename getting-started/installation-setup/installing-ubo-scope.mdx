# Installing uBO Scope

This guide provides clear, step-by-step instructions to install the uBO Scope browser extension on Chrome (Chromium-based browsers), Firefox, and Safari. Whether you prefer installing from official stores or via manual packaging, these instructions will ensure you get uBO Scope up and running quickly.

---

## 1. Installation Overview

uBO Scope is a lightweight companion extension that reports all network requests made during your browsing sessions. It supports the major modern browsers with tailored installation methods:

- **Chrome / Chromium**
- **Firefox**
- **Safari**

Each browser requires specific installation steps due to differing extension architectures and store availability.

---

## 2. Installing on Chrome / Chromium Browsers

### Option A: Install from Chrome Web Store

1. Open the Chrome Web Store page for uBO Scope: 
   [uBO Scope - Chrome Web Store](https://chromewebstore.google.com/detail/ubo-scope/bbdpgcaljkaaigfcomhidmneffjjjfgp)

2. Click **Add to Chrome**.

3. Confirm the extension permissions prompt.

4. After installation, the uBO Scope icon appears in the toolbar.

<Tip>
Ensure your browser version is Chrome 122 or later for compatibility.
</Tip>

### Option B: Manual Packaging and Installation

For advanced users or developers:

1. Clone or download the uBO-Scope repository.

2. Run the provided packaging script for Chromium:

```bash
./tools/make-package.sh chromium [version]
```

3. Load the unpacked extension:

- Open `chrome://extensions`
- Enable **Developer mode**
- Click **Load unpacked** and select the `build/uBO-Scope.chromium` directory

4. Confirm the extension appears in your toolbar.

<Warning>
Manually loaded unpacked extensions may not auto-update. Use store installation for seamless updates.
</Warning>

---

## 3. Installing on Firefox

### Option A: Install from Mozilla Add-ons

1. Navigate to the Firefox Add-ons page for uBO Scope:
   [uBO Scope - Firefox Add-ons](https://addons.mozilla.org/firefox/addon/ubo-scope/)

2. Click **Add to Firefox**.

3. Review and allow the permissions prompt.

4. The extension icon will appear in Firefox's toolbar.

<Tip>
Requires Firefox version 128 or newer.
</Tip>

### Option B: Creating and Installing the XPI Package Manually

1. Clone or download the uBO-Scope repository.

2. Use the build script to package for Firefox:

```bash
./tools/make-firefox.sh [version]
```

3. This generates an `.xpi` file in the `build` directory.

4. In Firefox, navigate to `about:addons` > gear icon > **Install Add-on From File...** and select the generated `.xpi`.

5. Confirm and install.

<Warning>
Manually installed add-ons may not auto-update. Installing from Mozilla Add-ons is recommended.
</Warning>

---

## 4. Installing on Safari

### Official Installation

1. Download uBO Scope from the Mac App Store or Safari Extensions Gallery if available.

2. Confirm the installation and grant required permissions.

3. The toolbar icon will appear for easy access.

<Tip>
Requires Safari 18.5 or later on macOS.
</Tip>

### Manual Installation

Safari extensions require packaging with Xcode and signing. Refer to Apple's documentation for Safari Web Extensions if manual installation is necessary.

---

## 5. Post-Installation Verification

1. After installation, open any webpage.

2. Look for the badge count on the uBO Scope icon reflecting the number of distinct third-party connections.

3. Click the icon to open the popup showing allowed, stealth-blocked, and blocked domains.

4. Confirm the popup loads without error and domains are visible.

<Check>
If badge count does not update or popup shows no data, check permissions and browser compatibility.
</Check>

## 6. Troubleshooting Common Installation Issues

<AccordionGroup title="Troubleshooting Installation Issues">
<Accordion title="Extension Icon Missing">
- Verify the extension is enabled in your browser’s extension/add-on manager.
- Restart the browser if needed.
- Confirm compatibility with your browser version.
</Accordion>

<Accordion title="Permissions Not Granted">
- Go to the extensions settings and ensure `webRequest`, `activeTab`, and `storage` permissions are approved.
- Some browsers require manual permission approval via prompt.
</Accordion>

<Accordion title="Popup Shows No Data">
- Ensure you have visited web pages that trigger network requests.
- Check if your content blockers or privacy tools are interfering with the extension.
- Refresh the page or restart the browser.
</Accordion>

<Accordion title="Manual Installation Issues">
- Confirm you used the correct packaging script for your platform.
- For unpacked installations, enable Developer Mode.
- Review browser console for any extension-related errors.
</Accordion>
</AccordionGroup>
