# Installing uBO Scope

This guide provides clear, step-by-step instructions to securely install the uBO Scope extension on supported browsers: Google Chrome, Mozilla Firefox, and Apple Safari. Follow the process tailored for your platform to ensure proper installation and readiness to start monitoring your browser's network connections.

---

## Supported Browsers and Minimum Versions

uBO Scope supports the following browsers and versions:

- **Google Chrome:** Version 122 or higher
- **Mozilla Firefox:** Version 128 or higher
- **Apple Safari:** Version 18.5 or higher

If you use an older version, please update your browser for compatibility.

---

## Installation Overview

For each supported browser, uBO Scope is available through official extension stores. Installing from these trusted sources guarantees authenticity and security.

The extension requires permissions to observe network requests, hence it will request authorization after installation.

---

## Step-by-Step Installation Instructions

### Google Chrome

1. Open Google Chrome (version 122+).
2. Visit the [Chrome Web Store listing for uBO Scope](https://chromewebstore.google.com/detail/ubo-scope/bbdpgcaljkaaigfcomhidmneffjjjfgp).
3. Click **Add to Chrome**.
4. In the permission prompt, review requested permissions carefully.
5. Select **Add extension** to complete the installation.
6. Confirm the uBO Scope icon appears near the address bar, indicating active installation.

<Check>
If the icon does not appear, check chrome://extensions/ and ensure uBO Scope is enabled.
</Check>

### Mozilla Firefox

1. Open Mozilla Firefox (version 128+).
2. Navigate to the [Firefox Add-ons page for uBO Scope](https://addons.mozilla.org/firefox/addon/ubo-scope/).
3. Click **Add to Firefox**.
4. Review requested permissions and select **Add**.
5. Once installed, you will see the uBO Scope icon near the address bar.

<Check>
If the icon is missing, verify via about:addons that uBO Scope is enabled.
</Check>

### Apple Safari

1. Open Safari (version 18.5+).
2. Safari extensions are installed via the **App Store** or Safari Extensions Gallery.
3. Visit the official source or Apple's Safari Extensions section and locate **uBO Scope**.
4. Install the extension and confirm it appears in Safari's toolbar.
5. Safari requires enabling the extension manually: go to **Safari > Preferences > Extensions** and check the box for uBO Scope.

<Note>
Safari installation may involve macOS system prompts; follow instructions carefully.
</Note>

---

## Post-Installation Steps

- **Verify extension activation:** Click the uBO Scope toolbar icon to open the popup interface and confirm it loads without errors.
- **Grant permissions:** If prompted, grant permissions for uBO Scope to observe network traffic.
- **Restart your browser:** This can resolve activation issues.

---

## Tips for a Secure Installation

- Always use official extension stores for installation to avoid tampered packages.
- Avoid sideloading or installing manually unless you are experienced and understand the source.
- Monitor permission requests carefully; uBO Scope only needs active tab, storage, and webRequest permissions.

## Troubleshooting Common Installation Issues

<AccordionGroup title="Troubleshooting Installation Issues">
<Accordion title="Extension Icon Does Not Appear">
- Check if the extension is enabled from your browser's extensions/settings page.
- Restart your browser after installation.
- Confirm you are using a supported browser version.
- For Chrome and Firefox, ensure the extension is not hidden behind the puzzle icon or in the overflow menu.
</Accordion>
<Accordion title="Permissions Not Granted or Network Monitoring Not Working">
- Make sure you allow all requested permissions during installation.
- Some network requests may not appear due to browser restrictions; see Compatibility & Integration Points guide.
</Accordion>
<Accordion title="Installation Blocked by Browser">
- Verify your browser settings allow installation of extensions from official stores.
- In enterprise or restricted environments, check with your admin about extension policies.
</Accordion>
</AccordionGroup>
