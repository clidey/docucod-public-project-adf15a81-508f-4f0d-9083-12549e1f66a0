# Installing uBO Scope

This guide provides clear, step-by-step instructions to install uBO Scope, a privacy-focused browser extension, on Chromium-based browsers, Firefox, and Safari. Follow the platform-specific instructions to add uBO Scope to your browser and verify the installation via the toolbar icon.

---

## 1. Supported Browsers and Requirements

Before installation, confirm your browser version meets these minimum requirements:

- **Chromium-based browsers** (Chrome, Edge, Brave, etc.): version 122 or later
- **Firefox**: version 128 or later
- **Safari**: version 18.5 or later

> uBO Scope requires access to the browser's `webRequest` API and the ability to display toolbar icons with badges.

---

## 2. Installation Methods

You can install uBO Scope officially through your browser’s extension store or sideload the extension manually if needed.

### 2.1 Official Extension Store Installation

| Browser          | Installation Link                                                                              |
|------------------|-----------------------------------------------------------------------------------------------|
| Chromium (Chrome) | [Chrome Web Store: uBO Scope](https://chromewebstore.google.com/detail/ubo-scope/bbdpgcaljkaaigfcomhidmneffjjjfgp) |
| Firefox          | [Firefox Add-ons: uBO Scope](https://addons.mozilla.org/firefox/addon/ubo-scope/)              |

### 2.2 Manual Installation (Developers or Testing)

For advanced users, the extension can be built from source and loaded as an unpacked extension.

#### Step-by-Step for Manual Load on Chromium:

1. Clone or download the [uBO Scope repository](https://github.com/gorhill/uBO-Scope).
2. Open a terminal, navigate to the project root.
3. Run the packaging script for your platform (Linux example):

```bash
./tools/make-package.sh chromium
```

4. Open your Chromium-based browser and navigate to `chrome://extensions/`.
5. Enable 'Developer mode' on the top right.
6. Click 'Load unpacked' and select the generated `build/uBO-Scope.chromium` directory.

#### Firefox Loading

1. Use `about:debugging` → 'This Firefox' → 'Load Temporary Add-on'.
2. Select `manifest.json` from the platform-specific directory (e.g., `platform/firefox/manifest.json`).

> **Note:** These manual steps are intended for development and testing — official releases offer smoother experience and auto-updates.

---

## 3. Installation per Browser

Follow the appropriate subsection below for your browser.

### 3.1 Installing on Chromium-Based Browsers

1. Open the Chrome Web Store link for uBO Scope.
2. Click **Add to Chrome**.
3. Review permissions and confirm installation.
4. The extension icon labeled **uBO Scope** will appear in the toolbar.

### 3.2 Installing on Firefox

1. Open the Firefox Add-ons link for uBO Scope.
2. Click **Add to Firefox**.
3. Approve permissions and install.
4. The extension icon will appear near the address bar.

### 3.3 Installing on Safari

1. Navigate to the Safari Extensions Gallery or App Store (from version 18.5 onwards).
2. Search for uBO Scope and install.
3. Enable the extension in Safari’s Preferences under Extensions.
4. The toolbar icon will appear once activated.

<Tip>
Safari’s extension support requires macOS 13 Ventura or later and Safari 18.5+. Make sure your system is updated.
</Tip>

---

## 4. Verifying Installation

Confirm the extension is active and ready:

1. Locate the **uBO Scope** icon in your browser’s toolbar.
2. Click the icon.
3. You should see a popup titled with the current tab’s domain and sections showing domains connected (allowed, stealth-blocked, blocked).
4. The icon badge will display a number representing the count of distinct third-party domains connected by the active tab.

<Check>
If the icon is missing or inactive:
- Verify the extension is enabled in your browser's extensions/add-ons management page.
- Refresh your browser or restart it if necessary.
- Ensure your browser version meets minimum requirements.
</Check>

## 5. Troubleshooting Common Installation Issues

<AccordionGroup title="Troubleshooting Installation Problems">
<Accordion title="Extension Icon Does Not Appear">
- Ensure permissions were granted during installation.
- Check if extension is blocked by browser security policies.
- Confirm browser supports manifest version 3 extensions.
</Accordion>

<Accordion title="Browser Not Supported or Version Too Low">
- Upgrade your browser to meet minimum version requirements (Chromium 122+, Firefox 128+, Safari 18.5+).
- Check browser release notes for `webRequest` API support.
</Accordion>

<Accordion title="Manual Installation Fails in Developer Mode">
- Confirm you selected the correct platform-specific `manifest.json` during load.
- Check console logs for permission-related errors.
- Ensure all files copied to the build folder, especially JS and manifest files.
</Accordion>
</AccordionGroup>

# Known Permissions

- `webRequest`: To monitor all network requests for third-party domains.
- `storage`: To save session and configuration data.
- `activeTab`: To interact with the current active tab's network details.
