# Install uBO Scope

Quickly install the uBO Scope extension on your preferred browser—Chrome, Firefox, or Safari—and start gaining transparent insights into your browser's third-party network connections.

---

## Supported Browsers and Minimum Versions

- **Google Chrome / Chromium**: Version 122 or later
- **Mozilla Firefox**: Version 128 or later
- **Safari**: Version 18.5 or later

uBO Scope requires these browser versions or newer due to dependencies on the `webRequest` API and modern Manifest V3 extension standards.

<Check>
Make sure your browser is updated to at least the minimum version to ensure compatibility and functionality.
</Check>

---

## Installation Overview

You can install uBO Scope from the official extension/add-on stores directly via the links below. After installation, the extension icon will appear in your toolbar, ready to provide real-time reporting.

---

## Step-by-Step Installation Instructions

### 1. Install on Google Chrome / Chromium

1. Open your Chrome or Chromium-based browser.
2. Navigate to the [Chrome Web Store page for uBO Scope](https://chromewebstore.google.com/detail/ubo-scope/bbdpgcaljkaaigfcomhidmneffjjjfgp).
3. Click **Add to Chrome**.
4. In the confirmation prompt, select **Add extension**.
5. Wait a few moments while the extension installs.

### 2. Install on Mozilla Firefox

1. Open Firefox browser.
2. Visit the [Firefox Add-ons page for uBO Scope](https://addons.mozilla.org/firefox/addon/ubo-scope/).
3. Click **Add to Firefox**.
4. Confirm by clicking **Add** in the permission prompt.
5. Allow the extension to install and activate.

### 3. Install on Safari

1. Open Safari browser on macOS or iOS supporting Safari 18.5+.
2. Open the **Safari Extensions** pane via Safari menu > Preferences > Extensions.
3. Search for “uBO Scope” or use the official Apple App Store link if available.
4. Click **Install** or **Enable** in Safari's extension management interface.

<Tip>
Because Safari extensions are distributed and installed differently than Chrome/Firefox, ensure you have permission to install extensions on your device via system preferences if you encounter issues.
</Tip>

---

## Confirming Successful Installation

After installation, verify uBO Scope is active:

1. Look for the uBO Scope icon in your browser toolbar; it looks like a shield with the letters “uBO.”
2. Click the icon to open the popup interface.
3. You should see a starting message such as “NO DATA” until you load a webpage.
4. Browse websites and observe the badge count update with the number of distinct third-party remote servers observed.

<Check>
If the icon does not appear:
- Ensure the extension is enabled in your browser's extension manager.
- Restart your browser and check again.
- For Safari, verify system extension permissions under macOS System Settings.
</Check>

---

## Additional Tips and Best Practices

- **Pin the Extension:** In browsers like Chrome and Firefox, pin uBO Scope to your toolbar for quick access and visibility.
- **Reload Open Tabs:** After installation, refresh your open tabs to allow uBO Scope to start monitoring and reporting connections.
- **Permissions:** uBO Scope requests standard permissions (`webRequest`, `activeTab`, `storage`) required to perform comprehensive network monitoring.

<Note>
The extension operates fully alongside any existing content blockers, ensuring accurate reporting of all network connections even if blockers are active.
</Note>

## Troubleshooting Common Installation Issues

<AccordionGroup title="Troubleshooting Installation">
<Accordion title="uBO Scope icon not visible after installation">
- Verify the extension is enabled in the browser’s extension/add-ons menu.

- Restart your browser.
- On Safari, check System Settings > Privacy & Security to allow extensions.
</Accordion>
<Accordion title="Unable to install due to browser version">
- Confirm your browser version meets the minimum requirements listed.
- Update your browser to the latest version.
</Accordion>
<Accordion title="Installation blocked by company or system policies">
- Contact your system administrator to allow extension installation and permissions.
</Accordion>
