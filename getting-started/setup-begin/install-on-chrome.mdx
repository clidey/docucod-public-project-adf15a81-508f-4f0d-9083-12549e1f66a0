# Install uBO Scope on Chrome

Welcome to your first step in discovering the detailed network connections your browser makes with uBO Scope. This guide will walk you through installing the extension from the Chrome Web Store, explain the permissions it requests, and help you open the popup interface to start monitoring connections immediately.

---

## 1. Prerequisites & System Requirements

- **Supported Chrome Versions**: Chrome 122 or higher is required to run uBO Scope due to its Manifest V3 compliance.
- **Operating Systems**: Compatible with Chrome on Windows, macOS, and Linux.
- **Permissions Needed**:
  - `activeTab`: Allows uBO Scope to interact with your current browser tab.
  - `storage`: Enables saving session data locally.
  - `webRequest`: Required to monitor and analyze network requests.

<Tip>
Ensure your browser is up to date to avoid installation or runtime issues with uBO Scope.
</Tip>

---

## 2. Installing uBO Scope from the Chrome Web Store

Follow these exact steps to install uBO Scope quickly and securely.

<Steps>
<Step title="Open the uBO Scope Chrome Web Store page">
Click this link to access the extension page: [uBO Scope on Chrome Web Store](https://chromewebstore.google.com/detail/ubo-scope/bbdpgcaljkaaigfcomhidmneffjjjfgp).
</Step>
<Step title="Click 'Add to Chrome'">
On the extension page, click the **Add to Chrome** button.
</Step>
<Step title="Confirm installation">
A dialog will appear showing the permissions requested. Review carefully:

- The extension will see data from your visited websites to monitor network connections.
- It needs to run in the background to track connections continuously.

Click **Add extension** to proceed.
</Step>
<Step title="Wait for installation to complete">
You will see a confirmation message once the extension is added. The uBO Scope icon will appear in your toolbar.
</Step>
</Steps>

<Note>
If the extension fails to install, ensure you are logged into your Google account and have access to the Chrome Web Store in your region.
</Note>

---

## 3. Quick Start: Opening and Using the uBO Scope Popup

Now that uBO Scope is installed, use the popup interface to view live connection data for your active tab.

<Steps>
<Step title="Locate the uBO Scope icon">
Look for the uBO Scope icon (a small shield) in your browser's toolbar, usually near the address bar.
</Step>
<Step title="Open the popup panel">
Click the icon once to open the popup window displaying network connection details.
</Step>
<Step title="Interpret the data">

- The popup shows the hostname of the active tab and summarizes connected domains.
- Domains are categorized as **not blocked**, **stealth-blocked**, or **blocked** to reflect network request outcomes.
- The badge on the icon shows the count of distinct third-party domains connected.

</Step>
</Steps>

<Check>
If the popup shows 'NO DATA' or empty results, reload the active tab and try opening the popup again.
</Check>

---

## 4. Tips for a Successful Installation and Early Use

- **Reload Tabs After Installation:** For uBO Scope to start monitoring, reload your open tabs to generate network data.
- **Monitor Badge Count:** A lower count on the toolbar badge means fewer distinct third-party connections, which is typically preferable.
- **Use with Content Blockers:** uBO Scope works independently and alongside your existing content blockers, offering unique visibility.

<Warning>
Permissions granted allow the extension to see all visited websites’ network requests. Only install uBO Scope from official sources to maintain security.
</Warning>

---

## 5. Troubleshooting Installation Issues

<Accordion title="Extension Does Not Appear After Installation">

- Check Chrome's extensions page (`chrome://extensions/`) to verify uBO Scope is enabled.
- Ensure your Chrome version meets the minimum requirement (122+).
- Restart Chrome and try installing again.

</Accordion>

<Accordion title="Popup Shows No Data or Empty Results">

- Reload the active browser tab to trigger new network requests.
- Confirm that the extension’s permissions were accepted fully.
- Check browser console for any errors (Right-click → Inspect → Console tab).

</Accordion>
