# Debunking Ad Blocker Myths with uBO Scope

## Overview
This guide empowers you to challenge common misconceptions surrounding content blockers using **uBO Scope**. By providing transparent, detailed insight into network requests and third-party connections, uBO Scope helps you see through misleading metrics like inflated block counts and unreliable ad blocker test pages. Instead of relying on block counts alone, this approach offers a more nuanced understanding of content blocker effectiveness and real network behavior.

---

## Why Debunking Matters

Many users assume that a content blocker with a high block count is inherently better, or that popular ad blocker test pages give reliable proof of effectiveness. These beliefs are incorrect and can lead to misguided decisions about privacy tools.

<u>**With uBO Scope, you gain clarity on:**</u>

- How block counts can be misleading by themselves
- Why test pages often fail to reflect real-world results
- The true impact on connections to third-party network servers

<Note>
The badge count on the uBO Scope icon represents the number of distinct third-party servers your browser contacted, _not_ the raw number of blocked requests. A **lower** count indicates better blocking of third-party connections.
</Note>

---

## Common Myths and How uBO Scope Debunks Them

### Myth 1: "Higher Block Count Means Better Blocking"

#### The Reality
A high number of blocked requests *does not* guarantee that fewer third-party connections are made. Sometimes, content blockers with high block counts still allow connections from more unique third-party servers.

#### How to Use uBO Scope

1. **Open uBO Scope on an active tab** in your browser.
2. Observe the **toolbar badge count**, which shows the number of distinct third-party domains connected.
3. Compare this count across different content blockers while keeping the same webpage loaded.
4. Analyze the **popup domain lists** under 'not blocked', 'stealth-blocked', and 'blocked' categories to see exactly which domains are allowed or blocked.

##### Success Indicator
When your content blocker truly blocks more third-party connections, the badge count will be lower, and the 'not blocked' list will have fewer domains.

---

### Myth 2: "Ad Blocker Test Pages Show Reliable Blocker Effectiveness"

#### The Reality
Ad blocker test pages:
- Often use unrealistic network requests to domains rarely or never encountered in real browsing.
- Fail to detect stealth techniques used by content blockers that prevent blocking detection by the webpage.

#### How uBO Scope Helps

Rather than trusting synthetic tests, uBO Scope shows real network activity during your everyday browsing:

- It collects information on network requests regardless of any content blocker or DNS-level blocking.
- You see which third-party servers your browser actually contacts or blocks on real pages.

## How to Use uBO Scope to Challenge Misleading Metrics

<Steps>
<Step title="Launch uBO Scope Popup">
Click the uBO Scope extension icon in your browser toolbar to open the popup displaying current tab network details.
</Step>
<Step title="Review Third-Party Connections">
Check the domains in the 'not blocked', 'stealth-blocked', and 'blocked' sections to understand which requests succeeded, which were blocked stealthily, and which were explicitly blocked.
</Step>
<Step title="Interpret Badge Count">
Use the toolbar icon’s badge count to see the total number of distinct third-party domains successfully connected to. Remember, lower counts indicate better blocking.
</Step>
<Step title="Compare Across Content Blockers">
Test the same webpage with different content blockers enabled and note differences in badge count and domain lists inside uBO Scope.
</Step>
<Step title="Analyze Stealth Blocking">
Pay special attention to the 'stealth-blocked' category; some blockers hide blocking activity to avoid detection, and uBO Scope reveals these stealth methods.
</Step>
</Steps>

## Practical Tips and Best Practices

- Use uBO Scope alongside your primary content blocker for independent verification of third-party connections.
- Don't rely solely on toolbar badge numbers reported by your blocker; instead, trust uBO Scope’s direct network observations.
- Regularly clear browsing data and retest to ensure consistent results.
- For filter list maintainers, use uBO Scope data to detect false positives/negatives and stealth blocking behavior.
- Understand nuances between blocked network requests and connections successfully avoided.

## Troubleshooting Common Challenges

<AccordionGroup title="Troubleshooting uBO Scope Insights">
<Accordion title="No Data Showing in uBO Scope Popup">
- Ensure the extension is properly installed and has the required permissions (activeTab, storage, webRequest).
- Refresh the tab and reopen the popup.
- Confirm your browser supports the required webRequest API.
- Close and reopen the tab or restart the browser if needed.
</Accordion>
<Accordion title="Badge Count Doesn't Match Expectations">
- Remember badge count reflects distinct third-party domains allowed, not total requests blocked.
- Blocked requests include retry attempts and redirects which inflate counts elsewhere.
- Cross-check with the popup to clarify which hosts contribute to the badge.
</Accordion>
<Accordion title="Stealth Blocking Not Detected by Test Pages">
- Some blockers implement stealth blocking to be undetectable by webpages.
- uBO Scope recognizes these by categorizing appropriately.
- Rely on uBO Scope data rather than test websites for accuracy.
</Accordion>
</AccordionGroup>

## Understanding Network Outcomes in uBO Scope

- **Not Blocked:** Domains from which requests succeeded.
- **Stealth Blocked:** Requests blocked silently to avoid detection.
- **Blocked:** Requests that were explicitly denied or resulted in errors.

This clarity allows you to differentiate aggressive blocking, passive allowance, and stealth techniques.
