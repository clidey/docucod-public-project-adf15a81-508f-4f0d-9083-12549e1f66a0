# Installing uBO Scope in Your Browser

This guide takes you through the installation of the uBO Scope browser extension across Chromium, Firefox, and Safari browsers. By following these steps, you'll enable real-time, transparent visibility into all third-party network connections initiated by webpages you visit—empowering you to better assess privacy and security.

---

## Workflow Overview

### Task Description
Install the uBO Scope extension safely and correctly on your preferred browser, along with tips to verify successful installation and usage.

### Prerequisites

- Supported browsers:
  - Chromium (version 122 or later)
  - Firefox (version 128 or later)
  - Safari (version 18.5 or later)
- Internet access to visit official browser extension marketplaces
- User permissions to add browser extensions

### Expected Outcome
You will have uBO Scope installed and accessible via your browser toolbar, ready to monitor and display third-party network connections on webpages.

### Time Estimate
5-10 minutes depending on browser and connection speed.

### Difficulty Level
Beginner to Intermediate

---

## Step-by-Step Installation Instructions

### 1. Identify Your Browser

Confirm which browser you intend to install uBO Scope on, as marketplace links and installation steps differ.

### 2. Access the Official Extension Marketplace

Use the following trusted sources for safe installation. Always avoid third-party sites to prevent tampering risks.

| Browser  | Installation Source                                                 |
|----------|-------------------------------------------------------------------|
| Chromium | [Chrome Web Store](https://chrome.google.com/webstore/category/extensions) (Search for 'uBO Scope') |
| Firefox  | [Firefox Add-ons](https://addons.mozilla.org/firefox/) (Search for 'uBO Scope')                      |
| Safari   | Use the Safari Extensions Gallery in the Mac App Store and search 'uBO Scope'                         |

<div className="tip">
Always verify the extension name is 'uBlock Origin Scope' by Raymond Hill to ensure authenticity.
</div>

### 3. Install from the Store

#### Chromium
- Open Chrome Web Store.
- Search for "uBO Scope".
- Select the extension authored by Raymond Hill.
- Click "Add to Chrome".
- Confirm permissions and accept to complete installation.

#### Firefox
- Open Firefox Add-ons website.
- Search for "uBO Scope".
- Select the extension authored by Raymond Hill.
- Click "Add to Firefox".
- Review permissions and approve installation.

#### Safari
- Launch App Store on your Mac.
- Navigate to the Safari Extensions category.
- Search for "uBO Scope".
- Click "Get" and then "Install".
- Safari will ask for permission to enable the extension, grant access to activate it.

### 4. Confirm Installation

- Look for the uBO Scope icon in your browser toolbar (usually a shield or the extension’s logo).
- The badge count will initially be empty or zero until you visit a webpage.

<div className="tip">
If the icon does not appear immediately, check your browser's extension or add-on manager to ensure it is enabled.
</div>

### 5. Verify uBO Scope is Active

- Open any website.
- Click the uBO Scope icon.
- A popup appears showing domains connected, categorized under ‘not blocked’, ‘stealth-blocked’, and ‘blocked’.
- The badge on the toolbar icon updates to show the total distinct connected third-party domains.

### 6. Troubleshooting Common Installation Issues

If you do not see expected behavior:
- Restart the browser.
- Ensure that the browser version meets the minimum required version.
- Clear browser cache.
- Reinstall uBO Scope from the official store.
- Check for browser extension conflicts or security restrictions.

For detailed troubleshooting, see the [Troubleshooting Common Setup Issues](https://your.documentation.link/getting-started/first-run-validation/troubleshooting-setup).

## Platform-Specific Considerations

| Feature/Requirement         | Chromium                               | Firefox                               | Safari                              |
|-----------------------------|--------------------------------------|-------------------------------------|------------------------------------|
| Minimum Version             | 122+                                 | 128+                                | 18.5+                              |
| Extension Source URL        | Chrome Web Store                     | Firefox Add-ons                     | Safari Extensions (Mac App Store)  |
| Background Script Type      | Service Worker (Module)               | Background Script (Module)           | Background Script (Module)          |
| Host Permissions            | http://, https://, ws://, wss://    | http://, https://, ws://, wss://   | http://, https://                  |
| Required Permissions        | activeTab, storage, webRequest       | activeTab, storage, webRequest      | activeTab, storage, webRequest      |

```json
// Example snippet from Chromium manifest.json
{
  "minimum_chrome_version": "122.0",
  "permissions": ["activeTab", "storage", "webRequest"],
  "host_permissions": [
    "https://*/*",
    "http://*/*",
    "wss://*/*",
    "ws://*/*"
  ]
}
```

<div className="note">
Safari currently does not support websocket network request monitoring through webRequest, so stealth blocking and related websocket events may be limited.
</div>
