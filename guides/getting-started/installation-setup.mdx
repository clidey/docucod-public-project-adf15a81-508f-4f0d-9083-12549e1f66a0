# Installing uBO Scope in Your Browser

This guide walks you through installing and verifying uBO Scope on supported browsers. You'll learn how to add the extension, access it, and check that it’s working correctly. Whether you use Chrome, Firefox, or other Chromium-based browsers, this tutorial gets you up and running within minutes.

---

## 1. Introduction

uBO Scope is a lightweight extension that reveals all remote server connections a webpage attempts or makes. It works alongside your content blockers to provide transparent insight into allowed, blocked, and stealth-blocked requests.

This guide focuses exclusively on installing uBO Scope, so you can start inspecting network connections quickly.

---

## 2. Prerequisites

Before you begin:

- **Supported Browsers:** Use one of the following:
  - Chromium-based browsers (Chrome, Edge, Opera, etc.) version 122.0 or later
  - Firefox version 128.0 or later
  
- **Permissions:** The extension requires permission to monitor network requests (`webRequest`), access active tabs, and store data.
- **Internet Connection:** Needed to install from official stores and fetch the extension.

Confirm your browser version meets the minimum requirement for your platform.

---

## 3. Installing on Chromium-Based Browsers (Chrome, Edge, etc.)

Follow these steps to install uBO Scope from the Chrome Web Store:

### Step 1: Open the Chrome Web Store uBO Scope page

- Visit the [uBO Scope Chrome Web Store page](https://chromewebstore.google.com/detail/ubo-scope/bbdpgcaljkaaigfcomhidmneffjjjfgp).

### Step 2: Click 'Add to Chrome'

- Click the "Add to Chrome" button.
- When prompted, confirm by selecting "Add extension".

### Step 3: Wait for installation

- The extension icon will appear in the toolbar.

### Step 4: Verify installation

- Right-click the icon and select "Manage extensions".
- Confirm the extension is enabled and has
  - Permissions: activeTab, storage, webRequest
  - Host permissions for `http://*/*` and `https://*/*`, plus `ws://*/*` and `wss://*/*` for WebSocket support.

### Step 5: Access the extension

- Click the uBO Scope icon on the toolbar to open its popup.

### Step 6: Confirm it is working

- Visit any webpage.
- Observe the badge number on the toolbar icon representing the count of distinct third-party remote servers connected.
- Click the icon to see detailed domain lists in the popup.

<Check>
If you don't see a badge or popup data, try refreshing the webpage or restarting the browser.
</Check>

## 4. Installing on Mozilla Firefox

Follow these steps to install uBO Scope from the Firefox Add-ons site:

### Step 1: Open Mozilla Add-ons page

- Go to the [uBO Scope Firefox Add-ons page](https://addons.mozilla.org/firefox/addon/ubo-scope/).

### Step 2: Click 'Add to Firefox'

- Click the "Add to Firefox" button on the page.
- Confirm any permission dialogs to proceed.

### Step 3: Confirm installation

- The uBO Scope icon appears in the Firefox toolbar.

### Step 4: Verify permissions

- Open the Add-ons Manager (`about:addons`).
- Ensure uBO Scope is enabled with these permissions:
  - Access your data for all websites (host permissions)
  - Monitoring web requests

### Step 5: Use the extension

- Visit any website.
- Note the badge count for connected third-party servers.
- Click the icon to view detailed domain connection information in the popup.

<Check>
If the badge is missing or no data appears, reload the tab or restart Firefox.
</Check>

---

## 5. Post-Installation Verification Steps

After installing, confirm uBO Scope functions properly using these steps:

### Visit a test webpage

- Load a popular website known to connect to multiple third-party servers (e.g., `https://example.com`).

### Check badge count

- The toolbar icon badge should show a number representing third-party domains detected.
- A higher number indicates more third-party connections.

### Open popup panel

- Click the toolbar icon.
- Review the sections:
  - **Not Blocked:** Domains the browser connected to successfully
  - **Stealth Blocked:** Domains where connection attempts were stealthily blocked
  - **Blocked:** Domains where connection attempts failed or were explicitly blocked

### Validate domain names

- Confirm the listed domains reflect expected third-party services (e.g., known CDNs or trackers).

### Troubleshoot

- If no data appears, ensure the extension is enabled.
- Check browser version compatibility.
- Refer to troubleshooting resources if issues persist.

---

## 6. Tips for a Successful Installation

- **Keep your browser updated**: Use the latest stable version to avoid compatibility issues.
- **Grant all permissions** the extension requests during installation.
- **Use official extension stores** only to avoid counterfeit versions.
- **Restart your browser** after installing to ensure services initialize correctly.
- **Visit multiple websites** to see varying badge counts and test data accuracy.

## 7. Common Issues and How to Fix Them

<AccordionGroup title="Common Installation Issues">
<Accordion title='The extension icon does not appear after installation'>
- Check if the extension is disabled in your browser’s extension manager.
- Make sure you installed the correct extension from official stores.
- Restart your browser and try again.
</Accordion>
<Accordion title='Badge count stays empty or at zero'>
- Visit a webpage that makes network requests to third-party servers.
- Confirm the extension has required permissions to monitor network requests.
- Disable conflicting privacy extensions temporarily to see if they interfere.
</Accordion>
<Accordion title='Popup panel shows "NO DATA" or empty sections'>
- Reload the active tab after installation.
- Restart the browser.
- If persistent, check for errors in browser console and report issues in the GitHub repository.
</Accordion>
</AccordionGroup>
