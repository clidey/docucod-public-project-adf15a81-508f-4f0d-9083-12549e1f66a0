# Installing and Setting Up uBO Scope

Welcome to your step-by-step guide for installing and setting up **uBO Scope**, the extension that reveals all remote server connections your browser makes. This guide covers installation on Chromium, Firefox, and Safari browsers, including minimum version requirements, trusted installation sources, and verification of successful setup.

---

## 1. Overview

uBO Scope is available as a browser extension designed for Chromium-based browsers, Firefox, and Safari. It uses browser WebRequest APIs to transparently track network requests, focusing on the number of distinct third-party remote servers contacted during web browsing.

This guide helps you:
- Install uBO Scope safely from official sources.
- Verify your browser meets the minimum compatibility requirements.
- Confirm the extension is installed and working correctly.

---

## 2. Prerequisites

Before installing uBO Scope, ensure the following prerequisites are met:

- **Supported Browsers and Minimum Versions:**
  - Chromium-based browsers (e.g., Google Chrome, Microsoft Edge): **version 122.0 or higher**
  - Mozilla Firefox: **version 128.0 or higher**
  - Safari: **version 18.5 or higher**

- **Permissions:** uBO Scope requires the browser permissions `activeTab`, `storage`, and `webRequest`. Additionally, it requires host permissions to access URLs with the schemes `http`, `https`, `ws`, and `wss`.

- **Internet connectivity** to fetch the extension from official stores and to allow uBO Scope to function properly.

- **Browser extension support for Manifest Version 3**, which all the supported browsers have at the required versions.

---

## 3. Installation Sources

Always install uBO Scope from the official and trusted extension stores to ensure authenticity and security:

| Browser     | Official Installation Source                                               |
|-------------|---------------------------------------------------------------------------|
| Chromium    | [Chrome Web Store](https://chromewebstore.google.com/detail/ubo-scope/bbdpgcaljkaaigfcomhidmneffjjjfgp) |
| Firefox     | [Firefox Add-ons](https://addons.mozilla.org/firefox/addon/ubo-scope/)   |
| Safari      | Safari Extensions Gallery (Apple's official App Store for Safari Extensions) |

**Do not install uBO Scope from unverified sources or third-party websites.**

---

## 4. Step-by-Step Installation Guide

<Steps>
<Step title="Install on Chromium (Chrome, Edge, etc.)">
1. Open your Chromium-based browser.
2. Navigate to the [Chrome Web Store uBO Scope page](https://chromewebstore.google.com/detail/ubo-scope/bbdpgcaljkaaigfcomhidmneffjjjfgp).
3. Click the "Add to Chrome" button.
4. When prompted, review and accept the permissions (`activeTab`, `storage`, `webRequest`, plus host permissions).
5. The extension icon will appear next to the address bar upon successful installation.
</Step>
<Step title="Install on Firefox">
1. Open Firefox.
2. Go to the [Firefox Add-ons uBO Scope page](https://addons.mozilla.org/firefox/addon/ubo-scope/).
3. Click "Add to Firefox".
4. Accept permissions requested by the extension.
5. Confirm installation.
6. The extension icon will show in the toolbar after installation.
</Step>
<Step title="Install on Safari">
1. Open Safari and open Safari Extensions from the menu.
2. Search for "uBO Scope" in the Safari Extensions Gallery.
3. Select the extension and click "Install."
4. Confirm any permission dialogs.
5. You may need to enable uBO Scope in Safari's Preferences > Extensions panel.
</Step>
</Steps>

## 5. Verifying Installation and Compatibility

After installing uBO Scope, follow these practical checks to verify proper setup:

### 5.1 Confirm Extension Presence

- Locate the uBO Scope icon in your browser’s toolbar (a shield-like icon with the uBO Scope logo).
- If missing, check browser extensions management interface to ensure it is enabled.

### 5.2 Confirm Browser Version

- Verify your browser is a supported version using these methods:
  - In Chromium, visit `chrome://settings/help`.
  - In Firefox, select "Help" > "About Firefox".
  - In Safari, choose "Safari" > "About Safari."

### 5.3 Test the Extension Functionality

- Visit any website that loads external content.
- Observe the toolbar badge on the uBO Scope icon; it shows the number of distinct third-party domains connected.
- Click the icon to open the popup interface that lists domains categorized as "not blocked," "stealth-blocked," and "blocked."

### 5.4 Check Permissions

- Make sure the extension has all required permissions:
  - For Chromium and Firefox: extensions management page.
  - For Safari: Extensions section of Safari preferences.

---

## 6. Platform-Specific Notes

### Chromium
- The minimum supported version is 122.0 due to Manifest Version 3 requirements.
- Permissions include access to URLs with HTTP, HTTPS, WS, and WSS schemes.

### Firefox
- Requires version 128.0 or newer.
- Has a Gecko-specific extension ID: `uBO-Scope@raymondhill.net`.
- Also requires `webRequest` and host permissions similar to Chromium.

### Safari
- Requires Safari 18.5 or newer.
- Permissions are slightly limited: only HTTP and HTTPS URLs are covered.
- The extension may need to be explicitly enabled after installation in Safari's preferences.

## 8. Troubleshooting Common Installation Issues

<AccordionGroup title="Troubleshooting Installation and Setup">
<Accordion title="Extension Icon Not Visible">
- Confirm the extension is enabled in your browser’s extensions page.
- For Chromium and Firefox, try restarting the browser.
- In Safari, check the Extensions tab in preferences and enable uBO Scope.
</Accordion>
<Accordion title="Permissions Not Granted">
- Reinstall the extension ensuring to accept all permission prompts.
- In some browsers, manually review and toggle permissions in extension settings.
</Accordion>
<Accordion title="Incompatible Browser Version">
- Confirm browser version meets or exceeds the minimum (Chromium 122.0+, Firefox 128.0+, Safari 18.5+).
- Upgrade your browser if below minimum.
</Accordion>
<Accordion title="Unexpected Badge Behavior">
- Reload the current tab or visit a different site.
- Ensure the site loads remote content (some pages may not have third-party requests).
- Restart the browser if the issue persists.
</Accordion>
</AccordionGroup>
