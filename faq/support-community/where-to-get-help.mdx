# Where can I get help or report issues?

This page is your guide to finding the right support channels and resources for uBO Scope. Whether you encounter installation problems, want to request new features, report bugs, or simply provide feedback, this documentation outlines the best ways to connect with the developer community and stay informed about updates.

---

## Frequently Asked Questions

### How can I get support if I face installation or usage issues?

- For **installation problems**, first consult the [Installing uBO Scope guide](../getting-started/installation-and-setup/installing-ubo-scope).
- If you encounter **unexpected behavior or issues**, check the [Troubleshooting Common Setup Issues](../getting-started/first-use-and-validation/troubleshooting-common-issues) guide.
- For advanced troubleshooting, refer to the [Troubleshooting and Common Issues guide](../../guides/realworld-usage/troubleshoot-common-issues).

If these resources do not resolve your problem, you can seek community or developer support by:

- Visiting the official GitHub repository’s [Issues page](https://github.com/gorhill/uBO-Scope/issues) to search existing issues or submit a new one.
- Engaging in discussions on forums or community channels related to uBlock Origin and browser privacy tools.


### Where can I submit feature requests or bug reports?

The preferred platform for reporting bugs or requesting features is the official GitHub repository:

- Use the [Issues page on GitHub](https://github.com/gorhill/uBO-Scope/issues) to:
  - File detailed bug reports describing the problem and steps to reproduce it.
  - Propose new features or enhancements.

Be sure to read existing issues to avoid duplicates and include relevant information such as your browser version, uBO Scope version, and a clear description of the problem or suggestion.


### How do I provide general feedback or learn about new releases?

- General feedback can be provided through GitHub discussions or by participating in community forums tied to uBlock Origin or privacy tools.
- Stay informed about new releases and updates by regularly visiting the [GitHub repository homepage](https://github.com/gorhill/uBO-Scope) or watching (subscribing to) the repository.


### What communication channels are available?

<u>Currently, uBO Scope’s support and development happen primarily through GitHub.</u> There are no official dedicated forums or chat channels maintained by the developer. Engaging on GitHub ensures your feedback reaches the developer and the community most directly.


## Best Practices for Submission

- **Clearly describe your issue or request:** Include steps to reproduce, observed behavior, expected behavior, and any relevant logs or screenshots.
- **Specify your environment:** Provide information about your browser type and version (e.g., Chrome 122+, Firefox 128+, Safari 18.5+), your uBO Scope extension version, and platform (Windows, macOS, Linux).
- **Search existing issues:** Before opening a new issue, confirm that it has not already been reported to avoid duplicates.


## Troubleshooting Tips Before Seeking Help

- Ensure the extension is up to date and enabled.
- Verify required permissions have been granted to uBO Scope.
- Reload the browser and revisit the affected sites.
- Check the popup panel and toolbar badge for expected reports.
- Consult the [Troubleshooting Common Setup Issues](../getting-started/first-use-and-validation/troubleshooting-common-issues) for quick fixes.
