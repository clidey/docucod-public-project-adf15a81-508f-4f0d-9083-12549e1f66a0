# Where Can I Get Support or Report Bugs?

This page guides you on the recommended channels to obtain help, submit bug reports, or ask questions related to uBO Scope. Whether you're encountering an issue, have a feature suggestion, or just need guidance, this document directs you to the most efficient and official avenues for support.

---

## Official GitHub Repository

The primary hub for reporting bugs, requesting features, and tracking development progress is the official uBO Scope GitHub repository.

- **Issues Page:**
  - Use this page to report new bugs or issues you encounter.
  - Before submitting, search existing issues to avoid duplicates.
  - Include as much detail as possible: browser version, uBO Scope version, steps to reproduce, screenshots, and any console logs.

  [uBO Scope GitHub Issues](https://github.com/gorhill/uBO-Scope/issues)

- **README and Wiki:**
  - The repository’s README file contains essential information and links.
  - Wiki pages or documentation folders offer detailed guides and troubleshooting tips.

- **Pull Requests:**
  - Developers and contributors can submit fixes or new features via pull requests.

> **Tip:** When reporting issues, provide a clear description and explicit steps to reproduce the problem to facilitate quicker resolution.

## Community and Discussion Forums

Although uBO Scope does not maintain a dedicated forum, users often discuss the extension and related topics in community spaces such as:

- **Reddit communities** focusing on privacy, ad blocking, and browser extensions.
- **GitHub Discussions** (if enabled in the repository) for informal questions and feature suggestions.
- **Related Projects' Forums:** Discussions on uBlock Origin or similar tools where uBO Scope may be mentioned.

Engaging with these communities can provide mutual assistance, tips, and best practice sharing.

---

## Support Guidelines

To get the best support experience:

- Always ensure you are using the latest version of uBO Scope and your browser.
- Try to reproduce issues with only uBO Scope enabled to rule out conflicts.
- Include detailed information as specified in GitHub issue templates.
- Be respectful and concise in your communication for better community engagement.

---

## Contacting the Author

Raymond Hill, the author of uBO Scope, maintains the project. Direct private support is limited, so GitHub Issues and community forums remain the best channels.
