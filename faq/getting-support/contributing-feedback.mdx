# How can I suggest improvements or contribute?

This page guides you on how to provide feedback, report bugs, request features, or contribute code to the uBO Scope project. It outlines the recommended process for engaging with the project through its official repository and highlights best practices to ensure your contributions are effective and well-received.

---

## Submitting Feedback and Feature Requests

If you have ideas to improve uBO Scope or want to request new features, the best place to communicate your suggestions is the project’s official GitHub repository:

- Visit the [uBO Scope GitHub repository](https://github.com/gorhill/uBO-Scope).
- Navigate to the **Issues** tab.
- Search existing issues to check if your suggestion has already been discussed or planned.
- If not found, open a new issue clearly describing your idea or improvement.

### Best Practices for Feedback

- Provide a clear and concise title summarizing your suggestion.
- Describe the enhancement or feature with context on why it is valuable.
- Include possible scenarios or use cases where this would improve your experience.
- Avoid vague or overly broad requests; specific proposals are easier to evaluate.

<Tip>
Construct your feature requests with real scenarios in mind—think about how it would help you or others achieve goals better or faster.
</Tip>

## Reporting Bugs

Encountering a problem? uBO Scope relies on community input to maintain quality and reliability.

- Check the [GitHub Issues](https://github.com/gorhill/uBO-Scope/issues) for similar reports.
- If none exists, open a new issue providing:
  - Steps to reproduce the problem.
  - The browser and version you are using (e.g., Chrome 122+, Firefox 128+, Safari 18.5+).
  - Any relevant logs, screenshots, or error messages you observed.

### Tips for Effective Bug Reports

- Be as detailed as possible in describing the issue.
- Mention your operating system and any other extensions running alongside uBO Scope.
- Describe what you expected to happen and what actually happened.

<Warning>
Incomplete bug reports slow down resolution. Providing detailed information upfront accelerates the process.
</Warning>

## Contributing Code and Changes

uBO Scope is an open-source project licensed under the GNU General Public License v3. Contributions to the source code are welcomed via the standard GitHub workflow.

### How to Contribute

1. **Fork the repository:**
   - Start by creating your own fork of [uBO Scope on GitHub](https://github.com/gorhill/uBO-Scope).
2. **Clone your fork locally:**
   - Work on issues or features by cloning your fork to your development environment.
3. **Develop your changes:**
   - Follow the project's coding guidelines and standards (ESLint configuration is available).
   - Ensure your code is clear, concise, and well-tested.
4. **Commit and push your changes:**
   - Use meaningful commit messages.
5. **Open a Pull Request (PR):**
   - Submit your PR against the `main` branch of the main repository.
   - Provide a detailed description of what your PR changes or fixes.

### Coding Standards and Practices

- The project uses strict JavaScript linting (see `eslint.config.mjs`) to maintain code quality.
- Your code should comply with these rules to avoid integration delays.
- Prefer clear and readable code over complex solutions.

### Review and Integration

- Project maintainers will review your pull request.
- You may be asked to clarify, refine, or expand your changes.
- When approved, your code will be merged and included in upcoming releases.

<Tip>
Familiarize yourself with the existing codebase, especially `js/popup.js`, `js/background.js`, and utilities in `js/ext.js` and `js/dom.js` to understand core workflows.
</Tip>

## Community and Support

- The GitHub repository issues page also acts as the main community forum for questions.
- Before opening a new issue, look for answers or open discussions that might help.

## Summary

By engaging through GitHub issues and pull requests, you help maintain and improve uBO Scope efficiently.
Following contribution guidelines ensures your feedback and code changes have the greatest positive impact.
